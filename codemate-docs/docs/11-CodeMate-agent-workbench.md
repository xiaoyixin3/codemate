# CodeMate 第八阶段：Agent 工作台

## 1. 阶段目标

第八阶段在现有 Thymeleaf 聊天页上增加可观察、可操作的 Agent 工作台，不引入新的前端框架，也不改变原有 WebSocket 聊天协议。

工作台解决以下问题：

- 用户能查看最近的 Agent Run 和当前运行状态；
- 流式回答期间能看到连接、接收和中断状态；
- 任务计划/工具步骤、工具结果摘要和引用证据集中展示；
- 失败原因使用可理解的中文提示；
- 非终态 Run 可以取消，失败或取消的 Run 可以重试；
- Bug 诊断仍通过原有二次确认入口保存为修复计划；
- 页面不展示工具调用参数或参数摘要。

## 2. 页面结构

聊天页右侧新增 `Agent 工作台`，包含四个区域：

1. 当前 Run：目标、模式、模型、状态、工具次数和 Token 预算；
2. 计划与工具步骤：步骤编号、工具名、状态、结果摘要和耗时；
3. 引用证据：文章标题、命中片段、文章链接和相关度；
4. 历史任务：最近 12 条 Agent Run，可点击切换。

桌面端默认展开；窄屏端使用右侧抽屉，保持聊天输入区域可用。

## 3. 接口复用

工作台复用第六阶段已有接口：

| 接口 | 用途 |
| --- | --- |
| `GET /agent-run/api?limit=12` | 最近 Run 列表 |
| `GET /agent-run/api/{runId}` | Run、步骤与证据详情 |
| `PUT /agent-run/api/{runId}/cancel` | 取消当前用户拥有的非终态 Run |

所有接口继续使用登录权限和当前用户 ID 进行数据隔离。前端对非终态 Run 进行短轮询，进入 `COMPLETED`、`FAILED` 或 `CANCELLED` 后停止。

## 4. 交互与错误反馈

- 发送请求：显示“等待模型响应”；
- 接收流：显示“正在接收流式回答”；
- 完整结束：显示“回答已完整接收”；
- WebSocket 中断：明确提示重连后重试；
- 接口网络异常：在工作台内显示具体加载/取消失败提示；
- 空检索：终态 Run 明确说明没有可引用的站内证据；
- 创建修复计划失败：保留原入口并恢复按钮，允许用户再次确认；
- 重试：将原 Run 的目标和模式填回聊天输入并重新发送，不复用旧 Run。

## 5. 安全边界

`AgentRunStepVo.argumentSummary` 不参与页面渲染。页面只展示后端提供的 `resultSummary`、`errorType` 和耗时，避免把检索词、内部参数或潜在敏感输入暴露到工作台。

引用链接使用固定的站内文章路径，并为新窗口链接设置 `noopener noreferrer`。所有动态文本先进行 HTML 转义。

## 6. 验证

- `node --check` 验证工作台脚本语法；
- MockMvc 覆盖 Run 列表、详情和取消接口，并校验当前用户 ID 透传；
- Playwright 覆盖桌面和手机视口、历史切换、详情渲染、失败重试、抽屉关闭、步骤、证据和参数隐藏；
- Maven 全量测试用于确认六模块回归。

浏览器自动化截图保存在 Codex 可视化输出目录，不进入项目源码。

## 7. 启动链加固

全量上下文测试额外发现并修复了前序阶段的两个容器启动问题：

- `ServiceAutoConfig` 补充扫描 memory、Agent Run 和 Bug diagnosis 的 MyBatis Mapper；
- `SafeToolExecutor`、`ToolMetrics` 明确标注生产构造器，避免存在测试构造器时 Spring 寻找无参构造器。

修复后应用上下文能完成 Agent 相关 Bean 装配，并继续执行到本地 MySQL 连接阶段。没有数据库凭据的环境中，两个既有数据库集成测试仍会按预期因无法连接 MySQL 而失败。
